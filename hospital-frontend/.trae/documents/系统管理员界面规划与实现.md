# 系统管理员界面规划与实现

## 一、功能模块规划

基于单医院管理系统的需求，管理员界面应包含以下功能模块：

1. **仪表盘**：系统概览、关键指标统计
2. **用户管理**：
   - 医生管理（新增、编辑、删除、查看）
   - 患者管理（查看、管理）
3. **科室管理**：科室信息管理
4. **排班管理**：医生排班设置
5. **挂号管理**：查看和管理所有挂号记录
6. **统计报表**：系统运行数据统计
7. **系统设置**：系统基础配置

## 二、技术实现方案

### 1. 布局设计
- 采用与医生界面一致的布局结构：左侧边栏 + 顶部导航栏 + 主内容区
- 组件复用：可复用部分布局组件（如TopNav），或根据需求调整

### 2. 路由设计
- 管理员路由前缀：`/admin/*`
- 主入口组件：`AdminApp.tsx`
- 页面组件：按功能模块划分

### 3. 组件结构
```
src/
├── components/
│   └── admin/          # 管理员专用组件
│       ├── AdminLayout.tsx    # 主布局组件
│       ├── Sidebar.tsx        # 侧边栏组件
│       ├── TopNav.tsx         # 顶部导航组件
│       └── AdminLayout.css    # 样式文件
├── pages/
│   └── admin/          # 管理员页面
│       ├── AdminApp.tsx       # 主入口组件
│       ├── Dashboard.tsx      # 仪表盘
│       ├── UserManagement.tsx # 用户管理
│       ├── DepartmentManagement.tsx # 科室管理
│       ├── ScheduleManagement.tsx # 排班管理
│       ├── RegistrationManagement.tsx # 挂号管理
│       ├── Statistics.tsx     # 统计报表
│       └── Settings.tsx       # 系统设置
└── routes/
    └── index.tsx       # 添加管理员路由
```

### 4. 实现步骤

1. **创建管理员布局组件**
   - 复制或参考DoctorLayout组件，创建AdminLayout
   - 设计管理员专属侧边栏菜单

2. **创建管理员主入口组件**
   - 实现AdminApp.tsx，配置路由
   - 使用useLocation钩子管理菜单激活状态

3. **实现功能页面**
   - 为每个功能模块创建对应的页面组件
   - 使用模拟数据填充页面
   - 实现基本的CRUD操作界面

4. **配置路由**
   - 在routes/index.tsx中添加管理员路由

5. **样式设计**
   - 设计管理员界面专属样式
   - 确保响应式设计

## 三、预期效果

- 管理员可以通过侧边栏菜单在不同功能模块间切换
- 每个功能页面都有模拟数据，展示完整的界面效果
- 布局美观，交互流畅
- 与现有系统风格保持一致

## 四、注意事项

1. 复用现有组件和样式，减少重复开发
2. 使用模拟数据，暂时不与后端交互
3. 确保代码结构清晰，便于后续扩展
4. 保持与现有系统的设计风格一致
