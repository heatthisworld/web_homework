# 医院挂号系统管理员功能模块文档

## 1. 概述

管理员是医院挂号系统的核心管理角色，负责系统的全面管理和维护，包括用户权限管理、基础数据维护、业务流程监控和统计分析等功能。

## 2. 功能模块划分

### 2.1 用户与权限管理

负责系统所有用户的管理，包括账号创建、权限分配和状态管理。

#### 2.1.1 用户列表

- **功能描述**：查看系统所有用户信息
- **操作权限**：查看
- **具体功能**：
  - 显示用户ID、用户名、角色、创建时间、状态
  - 支持按用户名、角色、状态进行搜索
  - 支持分页显示

#### 2.1.2 用户创建

- **功能描述**：创建新用户账号
- **操作权限**：创建
- **具体功能**：
  - 设置用户名、密码、角色（管理员/医生/患者）
  - 设置初始状态（启用/禁用）

#### 2.1.3 用户编辑

- **功能描述**：修改现有用户信息
- **操作权限**：编辑
- **具体功能**：
  - 修改用户名、密码、角色、状态
  - 查看用户操作日志

#### 2.1.4 用户删除

- **功能描述**：删除不需要的用户账号
- **操作权限**：删除
- **具体功能**：
  - 支持单个或批量删除
  - 删除前确认提示

### 2.2 基础数据管理

负责系统基础数据的维护，包括患者、医生、科室和病种信息。

#### 2.2.1 患者管理

- **功能描述**：管理患者基本信息
- **操作权限**：查看/创建/编辑/删除
- **具体功能**：
  - 患者信息列表（ID、姓名、性别、年龄、联系方式）
  - 支持按姓名、性别、年龄范围搜索
  - 创建/编辑患者基本信息
  - 删除患者（需确认无未完成挂号和病历）

#### 2.2.2 医生管理

- **功能描述**：管理医生信息和专业领域
- **操作权限**：查看/创建/编辑/删除
- **具体功能**：
  - 医生信息列表（ID、姓名、职称、科室、专长）
  - 支持按姓名、职称、科室搜索
  - 创建/编辑医生基本信息
  - 分配医生可诊治的病种
  - 删除医生（需确认无未完成挂号和病历）

#### 2.2.3 科室管理

- **功能描述**：管理医院科室信息
- **操作权限**：查看/创建/编辑/删除
- **具体功能**：
  - 科室列表（ID、名称、描述）
  - 创建/编辑科室信息
  - 删除科室（需确认无关联医生和病种）

#### 2.2.4 病种管理

- **功能描述**：管理疾病种类信息
- **操作权限**：查看/创建/编辑/删除
- **具体功能**：
  - 病种列表（ID、名称、所属科室、描述）
  - 支持按名称、科室搜索
  - 创建/编辑病种信息
  - 删除病种（需确认无关联挂号和病历）

### 2.3 业务流程管理

负责挂号、病历等核心业务流程的管理和监控。

#### 2.3.1 挂号管理

- **功能描述**：管理患者挂号信息
- **操作权限**：查看/编辑/取消/删除
- **具体功能**：
  - 挂号列表（ID、患者、医生、病种、预约时间、状态）
  - 支持按患者ID、医生ID、病种ID、状态、日期范围搜索
  - 修改挂号信息（医生、病种、预约时间）
  - 取消挂号
  - 删除无效挂号记录

#### 2.3.2 病历管理

- **功能描述**：管理患者病历信息
- **操作权限**：查看/编辑/删除
- **具体功能**：
  - 病历列表（ID、患者、医生、就诊日期、诊断结果）
  - 支持按患者ID、医生ID、就诊日期范围搜索
  - 查看病历详细信息（症状、诊断、用药、检查、治疗方案）
  - 编辑病历信息
  - 删除病历记录

### 2.4 统计分析

提供系统运营数据的统计和分析功能，帮助管理员了解系统使用情况。

#### 2.4.1 挂号统计

- **功能描述**：统计挂号业务数据
- **操作权限**：查看
- **具体功能**：
  - 按日/周/月统计挂号数量
  - 按科室统计挂号分布
  - 按医生统计挂号量
  - 按状态统计挂号情况

#### 2.4.2 就诊统计

- **功能描述**：统计患者就诊数据
- **操作权限**：查看
- **具体功能**：
  - 按日/周/月统计就诊人数
  - 按病种统计就诊分布
  - 平均就诊时长统计

#### 2.4.3 系统使用统计

- **功能描述**：统计系统整体使用情况
- **操作权限**：查看
- **具体功能**：
  - 用户注册量统计
  - 活跃用户统计
  - 系统访问量统计

### 2.5 系统设置

负责系统基础参数和配置的管理。

#### 2.5.1 系统参数设置

- **功能描述**：设置系统运行参数
- **操作权限**：设置
- **具体功能**：
  - 挂号时间范围设置
  - 预约提前天数设置
  - 系统维护公告设置

#### 2.5.2 数据备份与恢复

- **功能描述**：管理系统数据备份
- **操作权限**：备份/恢复
- **具体功能**：
  - 手动触发数据备份
  - 查看备份历史
  - 从备份恢复数据

#### 2.5.3 日志管理

- **功能描述**：查看系统操作日志
- **操作权限**：查看
- **具体功能**：
  - 操作日志列表（用户、操作类型、操作时间、IP地址）
  - 支持按用户、操作类型、时间范围搜索
  - 导出日志数据

## 4. 权限设计建议

### 4.1 角色权限矩阵

| 功能模块    | 管理员 | 医生       | 患者   |
| ------- | --- | -------- | ---- |
| 用户与权限管理 | 全部  | 无        | 无    |
| 患者管理    | 全部  | 查看       | 查看自己 |
| 医生管理    | 全部  | 查看自己     | 无    |
| 科室管理    | 全部  | 查看       | 无    |
| 病种管理    | 全部  | 查看       | 无    |
| 挂号管理    | 全部  | 查看/处理    | 查看自己 |
| 病历管理    | 全部  | 查看/创建/编辑 | 查看自己 |
| 统计分析    | 全部  | 部分       | 无    |
| 系统设置    | 全部  | 无        | 无    |

### 4.2 操作权限级别

1. **查看权限**：只能查看数据，不能进行修改
2. **创建权限**：可以创建新数据
3. **编辑权限**：可以修改现有数据
4. **删除权限**：可以删除数据
5. **管理权限**：可以进行所有操作，包括权限分配

## 5. 前端实现建议

### 5.1 界面设计建议

- **左侧导航**：按功能模块分组显示
- **顶部导航**：用户信息、消息通知、退出登录
- **主内容区**：数据列表、表单、统计图表
- **响应式设计**：适配不同屏幕尺寸

### 5.2 用户体验建议

- **批量操作**：支持批量删除、批量修改
- **搜索过滤**：提供多条件搜索功能
- **分页显示**：大数据量时分页处理
- **操作确认**：重要操作前进行确认提示
- **加载状态**：数据加载时显示加载动画

## 6. 与后端API对接(暂时不对接,前端直接生成数据展示样式)

管理员功能模块需要调用的主要后端API包括：

1. **用户管理API**：
   
   - GET /api/users - 获取用户列表
   - POST /api/users - 创建用户
   - PUT /api/users/{id} - 更新用户
   - DELETE /api/users/{id} - 删除用户

2. **患者管理API**：
   
   - GET /api/patients - 获取患者列表
   - POST /api/patients - 创建患者
   - PUT /api/patients/{id} - 更新患者
   - DELETE /api/patients/{id} - 删除患者

3. **医生管理API**：
   
   - GET /api/doctors - 获取医生列表
   - POST /api/doctors - 创建医生
   - PUT /api/doctors/{id} - 更新医生
   - DELETE /api/doctors/{id} - 删除医生

4. **病种管理API**：
   
   - GET /api/diseases - 获取病种列表
   - POST /api/diseases - 创建病种
   - PUT /api/diseases/{id} - 更新病种
   - DELETE /api/diseases/{id} - 删除病种

5. **挂号管理API**：
   
   - GET /api/registrations - 获取挂号列表
   - POST /api/registrations - 创建挂号
   - PUT /api/registrations/{id} - 更新挂号
   - DELETE /api/registrations/{id} - 删除挂号

6. **病历管理API**：
   
   - GET /api/medical-records - 获取病历列表
   - POST /api/medical-records - 创建病历
   - PUT /api/medical-records/{id} - 更新病历
   - DELETE /api/medical-records/{id} - 删除病历
