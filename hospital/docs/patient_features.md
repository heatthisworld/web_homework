# 医院挂号系统患者功能模块文档

## 1. 概述
患者是医院挂号系统的核心用户之一，主要通过系统完成在线挂号、查看挂号记录、管理个人信息和查看病历等功能。本文档详细描述了患者在系统中可以使用的所有功能模块。

## 2. 功能模块划分

### 2.1 用户认证与个人中心
患者的基础功能，包括注册、登录和个人信息管理。

#### 2.1.1 用户注册
- **功能描述**：新患者注册系统账号
- **操作权限**：所有未注册用户
- **具体功能**：
  - 输入个人基本信息（姓名、身份证号、性别、年龄、联系方式等）
  - 设置登录用户名和密码
  - 阅读并同意用户协议
  - 完成注册并自动登录

#### 2.1.2 用户登录
- **功能描述**：已注册患者登录系统
- **操作权限**：所有已注册用户
- **具体功能**：
  - 输入用户名和密码
  - 忘记密码找回功能
  - 记住登录状态选项
  - 登录后跳转到个人中心

#### 2.1.3 个人中心
- **功能描述**：患者查看和管理个人信息
- **操作权限**：已登录患者
- **具体功能**：
  - 查看个人基本信息（姓名、性别、年龄、联系方式等）
  - 修改个人信息（除身份证号外的其他信息）
  - 修改登录密码
  - 绑定/解绑手机号码
  - 查看账户安全设置

### 2.2 科室与医生查询
患者查找合适的科室和医生的功能。

#### 2.2.1 科室列表
- **功能描述**：查看医院所有科室信息
- **操作权限**：已登录患者
- **具体功能**：
  - 显示科室名称、简介、主治病种
  - 支持按科室名称搜索
  - 点击科室查看该科室下的医生列表

#### 2.2.2 医生列表
- **功能描述**：查看指定科室的医生信息
- **操作权限**：已登录患者
- **具体功能**：
  - 显示医生姓名、职称、专长、简介、可挂号时间
  - 支持按医生姓名、职称搜索
  - 查看医生详细信息

#### 2.2.3 医生详情
- **功能描述**：查看医生的详细信息
- **操作权限**：已登录患者
- **具体功能**：
  - 医生基本信息（姓名、职称、所属科室、学历、工作年限等）
  - 医生专长和简介
  - 医生可挂号的时间段
  - 患者评价和评分

### 2.3 挂号功能
患者进行在线挂号的核心功能。

#### 2.3.1 挂号流程
- **功能描述**：患者完成挂号的完整流程
- **操作权限**：已登录患者
- **具体流程**：
  1. 选择科室
  2. 选择医生
  3. 选择就诊日期和时间段
  4. 确认挂号信息
  5. 完成挂号

#### 2.3.2 选择就诊时间
- **功能描述**：选择具体的就诊日期和时间段
- **操作权限**：已登录患者
- **具体功能**：
  - 日历形式展示可挂号日期
  - 显示每个日期下的可挂号时间段
  - 标记已约满的时间段
  - 选择合适的时间段进行挂号

#### 2.3.3 确认挂号信息
- **功能描述**：确认挂号的所有信息
- **操作权限**：已登录患者
- **具体功能**：
  - 显示患者信息、选择的科室、医生、就诊时间
  - 显示挂号费用（如有）
  - 确认无误后提交挂号申请

#### 2.3.4 挂号成功
- **功能描述**：挂号成功后的反馈
- **操作权限**：已登录患者
- **具体功能**：
  - 显示挂号成功提示
  - 生成挂号凭证和唯一挂号ID
  - 提示就诊注意事项
  - 可选择保存或打印挂号凭证

### 2.4 挂号记录管理
患者查看和管理自己的挂号记录。

#### 2.4.1 挂号记录列表
- **功能描述**：查看所有挂号记录
- **操作权限**：已登录患者
- **具体功能**：
  - 显示所有挂号记录，包括待就诊、已就诊、已取消的记录
  - 每条记录显示挂号ID、科室、医生、就诊时间、挂号时间、状态
  - 支持按时间段、科室、状态筛选
  - 支持分页显示

#### 2.4.2 挂号记录详情
- **功能描述**：查看挂号记录的详细信息
- **操作权限**：已登录患者
- **具体功能**：
  - 显示完整的挂号信息
  - 显示挂号凭证
  - 显示就诊医生的联系方式（如有）
  - 显示就诊地点和注意事项

#### 2.4.3 取消挂号
- **功能描述**：取消已预约但未就诊的挂号
- **操作权限**：已登录患者
- **具体功能**：
  - 仅可取消未就诊的挂号记录
  - 取消前显示确认提示
  - 取消成功后更新挂号状态
  - 发送取消通知（如有）

### 2.5 病历管理
患者查看自己的病历信息。

#### 2.5.1 病历列表
- **功能描述**：查看所有病历记录
- **操作权限**：已登录患者
- **具体功能**：
  - 显示所有病历记录，按就诊时间倒序排列
  - 每条记录显示就诊日期、就诊科室、主治医生、诊断结果
  - 支持按时间段、科室筛选
  - 支持分页显示

#### 2.5.2 病历详情
- **功能描述**：查看病历的详细内容
- **操作权限**：已登录患者
- **具体功能**：
  - 显示完整的病历信息，包括：
    - 就诊日期和时间
    - 就诊科室和医生
    - 症状描述
    - 诊断结果
    - 用药记录
    - 检查结果
    - 治疗方案
    - 医生备注
  - 支持查看历史病历的对比

### 2.6 系统通知与帮助
患者获取系统通知和使用帮助的功能。

#### 2.6.1 系统通知
- **功能描述**：查看系统发送的通知消息
- **操作权限**：已登录患者
- **具体功能**：
  - 显示所有系统通知，包括：
    - 挂号成功通知
    - 就诊提醒
    - 系统维护通知
  - 支持按时间排序查看
  - 标记已读/未读状态

#### 2.6.2 使用帮助
- **功能描述**：查看系统使用指南和常见问题
- **操作权限**：所有用户
- **具体功能**：
  - 提供系统使用流程图
  - 常见问题解答（FAQ）
  - 联系客服的方式

## 3. 权限设计

### 3.1 患者权限矩阵

| 功能模块         | 患者权限 |
|------------------|----------|
| 用户注册         | 完全访问 |
| 用户登录         | 完全访问 |
| 个人中心         | 查看/编辑自己的信息 |
| 科室与医生查询   | 完全访问 |
| 挂号功能         | 完全访问 |
| 挂号记录管理     | 查看/管理自己的记录 |
| 病历管理         | 查看自己的病历 |
| 系统通知与帮助   | 完全访问 |

### 3.2 数据访问限制

1. **个人信息**：患者只能查看和编辑自己的个人信息
2. **挂号记录**：患者只能查看和管理自己的挂号记录
3. **病历信息**：患者只能查看自己的病历信息
4. **医生信息**：患者只能查看医生的公开信息，不能编辑
5. **科室信息**：患者只能查看科室信息，不能编辑

## 4. 前端实现建议

### 4.1 界面设计原则

- **简洁直观**：界面设计简洁明了，减少患者的学习成本
- **流程引导**：通过清晰的流程引导患者完成挂号等操作
- **响应式设计**：适配不同设备（手机、平板、电脑）
- **信息清晰**：重要信息（如挂号时间、地点）突出显示
- **操作便捷**：常用功能一键可达，减少点击次数

### 4.2 界面布局建议

#### 4.2.1 移动端布局
- **底部导航栏**：主要功能入口（首页、挂号、记录、我的）
- **顶部状态栏**：搜索、消息通知
- **主内容区**：功能模块的具体内容

#### 4.2.2 PC端布局
- **左侧导航栏**：主要功能模块列表
- **顶部导航栏**：用户信息、消息通知、搜索
- **主内容区**：功能模块的具体内容

### 4.3 用户体验优化

1. **操作反馈**：所有操作都应有明确的反馈（如按钮点击、提交成功等）
2. **错误提示**：提供清晰的错误提示和解决建议
3. **加载状态**：数据加载时显示加载动画，避免患者等待焦虑
4. **表单验证**：实时验证表单输入，减少提交错误
5. **就诊提醒**：通过消息、短信等方式提醒患者就诊时间

## 5. 与后端API对接

患者功能模块需要调用的主要后端API包括：

1. **用户认证API**：
   - POST /api/auth/register - 用户注册
   - POST /api/auth/login - 用户登录
   - POST /api/auth/logout - 用户登出
   - POST /api/auth/forgot-password - 忘记密码

2. **个人中心API**：
   - GET /api/patients/profile - 获取个人信息
   - PUT /api/patients/profile - 更新个人信息
   - PUT /api/patients/password - 修改密码

3. **科室与医生API**：
   - GET /api/departments - 获取科室列表
   - GET /api/doctors - 获取医生列表
   - GET /api/doctors/{id} - 获取医生详情

4. **挂号API**：
   - GET /api/registrations/available-times - 获取可挂号时间
   - POST /api/registrations - 创建挂号
   - GET /api/registrations/patient - 获取患者的挂号记录
   - GET /api/registrations/{id} - 获取挂号详情
   - PUT /api/registrations/{id}/cancel - 取消挂号

5. **病历API**：
   - GET /api/medical-records/patient - 获取患者的病历记录
   - GET /api/medical-records/{id} - 获取病历详情

6. **通知API**：
   - GET /api/notifications - 获取通知列表
   - PUT /api/notifications/{id}/read - 标记通知为已读

## 6. 功能流程示例

### 6.1 挂号流程示例

1. 患者登录系统
2. 在首页或科室列表中选择科室
3. 查看该科室下的医生列表，选择合适的医生
4. 查看医生的可挂号时间，选择就诊日期和时间段
5. 确认挂号信息（患者信息、科室、医生、就诊时间）
6. 提交挂号申请
7. 系统显示挂号成功，并生成挂号凭证
8. 患者可以在挂号记录中查看详细信息

### 6.2 查看病历流程示例

1. 患者登录系统
2. 进入"我的病历"页面
3. 查看所有病历记录列表
4. 点击某个病历记录，查看详细内容
5. 可以查看症状、诊断、用药、检查结果等信息

## 7. 总结

患者功能模块是医院挂号系统的重要组成部分，直接影响患者的就医体验。本文档详细描述了患者在系统中可以使用的所有功能模块，包括用户认证、个人中心、科室与医生查询、挂号功能、挂号记录管理、病历管理以及系统通知与帮助等。

前端开发人员可以参考本文档进行患者端的界面设计和功能实现，确保患者能够方便、快捷地使用系统完成各项操作。
